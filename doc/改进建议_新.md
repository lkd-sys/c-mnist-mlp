我们有一个用C语言实现的MLP（多层感知机）来处理MNIST数据集的项目。项目包含多个文件，分别实现了数据加载、前向传播、反向传播、参数更新、模型保存和测试等功能。以下是对代码的总体观察和一些改进建议：
# 总体观察
1. **代码组织**：代码被分为多个文件，每个文件负责一个功能模块，这有助于维护。
2. **内存管理**：动态内存分配被广泛使用，需要注意释放内存以避免泄漏。
3. **激活函数**：使用了带泄露的ReLU（Leaky ReLU）作为隐藏层的激活函数，输出层使用Softmax。
4. **损失函数**：在训练时使用交叉熵损失（在反向传播中体现），在测试时使用MSE（平均绝对误差）作为打印的损失指标。但注意，在训练时，反向传播使用的是交叉熵的梯度，而打印的损失是平均绝对误差（L1损失）。这可能不是最优，因为训练指标和梯度下降的损失函数==不一致==。
5. **初始化**：权重和偏置使用随机初始化，但权重初始化为0.1乘以[0,1)之间的随机数（即均匀分布），这可能导致初始化值过小。通常==建议使用Xavier或He初始化==。
# 改进建议
## 1. 损失函数的计算
在`train_acc_param_avg_abs.c`中，训练损失计算为平均绝对误差（L1损失），而在`test_acc_loss.c`中测试损失也是平均绝对误差。但是，在反向传播中（`backward.c`）使用的是交叉熵损失的梯度（Softmax输出后的交叉熵）。虽然这样也能工作，但通常我们使用交叉熵损失作为分类问题的损失函数，并且打印交叉熵损失可以更好地反映模型训练的情况。建议将训练和测试的损失函数都改为交叉熵损失（或至少打印交叉熵损失），这样与反向传播一致。
例如，在`forward.c`中已经计算了Softmax输出，那么交叉熵损失可以这样计算：
```c
double ce_loss = 0.0;
for (int j = 0; j < OUTPUT_LAYER_SIZE; j++) {
    if (j == train_labels[(img_tag + i) % TRAIN_IMG_NUM]) {
        ce_loss -= log(p[i][j] + 1e-8); // 防止log(0)
    }
}
train_loss += ce_loss; // 累积交叉熵损失
```
然后在`train_acc_param_avg_abs.c`中计算整个批次的平均交叉熵损失。同样地，在测试函数中也可以改为计算交叉熵损失。
## 2. 权重初始化
在`model_initializer.c`中，权重初始化为0.1乘以[0,1)之间的随机数（均匀分布）。这种初始化方法可能会导致梯度消失或爆炸。对于使用ReLU的网络，推荐使用He初始化（方差为2/n的均匀分布或正态分布）。例如，对于某一层，输入节点数为`n`，则权重初始化为从均匀分布`[-sqrt(6/n), sqrt(6/n)]`中采样（He uniform）或者正态分布（均值为0，方差为`sqrt(2/n)`）。
改进示例（针对某一层的权重）：
```c
// He initialization for ReLU
double limit = sqrt(6.0 / layer_array[i]); // 对于均匀分布
w[i][j][k] = (double)rand() / RAND_MAX * 2 * limit - limit;
```
注意：由于使用了Leaky ReLU，可以尝试使用He初始化，或者针对Leaky ReLU的初始化方法（如将方差改为`sqrt(2/(1+alpha^2)/n)`，其中alpha是Leaky ReLU的负斜率）。
## 3. 学习率调整
目前学习率是固定的，但实践中学习率衰减可以帮助模型更好地收敛。可以考虑在训练过程中逐渐降低学习率。
例如，在`parameter_update.c`中，可以在每个epoch（或每一定epoch）后对学习率进行衰减：
```c
// 例如，每1000个epoch将学习率乘以0.9
if (epoch % 1000 == 0) {
    for (int i = 0; i < HIDDEN_LAYER_COUNT + 1; i++) {
        learning_rate[0][i] *= 0.9;
        learning_rate[1][i] *= 0.9;
    }
}
```
## 4. 梯度裁剪
在反向传播中，梯度可能会变得很大，导致训练不稳定。梯度裁剪可以防止梯度爆炸。
在`backward.c`中，计算完梯度后，在更新参数之前，可以对梯度进行裁剪。例如：
```c
// 梯度裁剪（在更新参数之前）
double max_grad = 1.0; // 设置一个阈值
for (int i = 0; i < HIDDEN_LAYER_COUNT + 1; i++) {
    for (int j = 0; j < layer_array[i + 1]; j++) {
        if (dz[i][j] > max_grad) dz[i][j] = max_grad;
        else if (dz[i][j] < -max_grad) dz[i][j] = -max_grad;
        for (int k = 0; k < layer_array[i]; k++) {
            if (dw[i][j][k] > max_grad) dw[i][j][k] = max_grad;
            else if (dw[i][j][k] < -max_grad) dw[i][j][k] = -max_grad;
        }
    }
}
```
## 5. 内存泄漏
在`test_acc_loss`和`backward`函数中，使用`malloc`分配了内存，但在某些路径下（例如提前返回）可能会忘记释放。确保所有分配的内存都被释放。
例如，在`test_acc_loss`函数中，在函数末尾有`free(p1); free(p2);`，这是好的。但在`backward`函数中，同样有`free(p1); free(p2);`，也没有问题。但是，请注意在`backward`函数中，如果中间出现错误需要提前返回，则可能会跳过释放步骤。目前这些函数中没有提前返回，所以是安全的。
## 6. 数值稳定性
在计算Softmax时，指数函数可能导致数值溢出。通常的做法是减去最大值：
```c
double max_val = z[HIDDEN_LAYER_COUNT][i][0];
for (int j = 1; j < OUTPUT_LAYER_SIZE; j++) {
    if (z[HIDDEN_LAYER_COUNT][i][j] > max_val) max_val = z[HIDDEN_LAYER_COUNT][i][j];
}
double sum = 0.0;
for (int j = 0; j < OUTPUT_LAYER_SIZE; j++) {
    e_z[j] = exp(z[HIDDEN_LAYER_COUNT][i][j] - max_val);
    sum += e_z[j];
}
```
在`forward.c`和`test_acc_loss.c`中，计算Softmax时都没有减去最大值，这可能在数值较大时导致`exp`溢出。建议修改。
## 7. 测试集评估
在`main.c`中，每100个epoch在测试集上评估一次。但是，测试集评估比较耗时，如果训练时间很长，可以适当减少评估频率（比如每1000个epoch）。同时，在保存模型时，目前是根据测试损失是否比上一次保存的测试损失小来决定是否保存。但要注意，测试损失可能会波动，所以也可以考虑使用验证集来保存最佳模型。
## 8. 批量归一化（Batch Normalization）
==批量归一化可以加速训练并提高模型性能==。==考虑在每个隐藏层的激活函数前加入批量归一化层==。
## 9. 代码可读性和维护
- 在`data_loader.c`中，读取路径的方式比较原始。可以考虑使用更安全的字符串操作函数（如`strncat`）来拼接路径。
- 在多个文件中，使用魔数（magic number）进行文件路径拼接（如添加字符串"/train-images.idx3-ubyte"），建议将这些字符串定义为常量。
## 10. 错误处理
在`data_loader.c`中，有基本的错误处理（如文件打开失败），但在其他文件中，错误处理较少。例如，在`model_saver.c`中，打开文件失败时应该处理错误。
# 总结
以上建议主要围绕损失函数、初始化、学习率、梯度处理、数值稳定性和代码质量等方面。改进这些点可能会提高训练效率、模型性能和代码健壮性。由于项目较大，建议逐步进行改进和测试。
