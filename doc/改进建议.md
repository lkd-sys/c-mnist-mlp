我们有一个多层感知机（MLP）项目，用于处理MNIST数据集。代码组织良好，但有一些地方可以改进。以下是一些建议：
1. **内存管理**：
   - 在`backward.c`和`test_acc_loss.c`中，每次调用函数时都使用`malloc`和`free`分配和释放临时数组（p1, p2）。这可能会引起性能开销，尤其是在训练循环中频繁调用`backward`函数。我们可以考虑将这些数组作为静态变量分配一次，或者在全局状态中分配，以避免重复分配和释放。
2. **数值稳定性**：
   - 在`forward.c`和`test_acc_loss.c`中，计算softmax时，直接计算`exp(z)`可能会导致数值溢出（尤其是当z很大时）。通常的做法是减去最大值：`exp(z[i] - max_z)`，其中`max_z`是z数组中的最大值。
3. **激活函数**：
   - 当前使用Leaky ReLU，但代码中多处直接使用`ACT_FUNC`宏，而在反向传播中需要手动计算导数。可以考虑将激活函数及其导数封装成函数，以提高代码可读性和可维护性。
4. **梯度累加**：
   - 在`backward.c`中，梯度是通过累加（+=）计算的。注意在每次batch开始时，需要将梯度（dw, dz）重置为0。目前，`reset`函数在每次迭代后被调用（在`main.c`中），这是正确的。但要注意在`backward`函数中，梯度累加之前，`dw`和`dz`应该已经被重置为0。
5. **学习率调整**：
   - 目前学习率是固定的。可以考虑实现学习率衰减，例如随着epoch增加而逐渐减小学习率。
6. **模型保存与加载**：
   - `model_saver.c`中，文件路径是硬编码的。最好将其作为参数或定义在配置文件中。
7. **错误处理**：
   - 在文件操作（如`data_loader.c`）中，有错误处理，但可以更详细（例如打印错误原因）。
8. **性能**：
   - 循环嵌套较深，可以考虑优化循环顺序以利用局部性（cache友好）。
   - 在`backward`函数中，内层循环有多个条件判断（如`if (j == HIDDEN_LAYER_COUNT)`），这些判断在循环内部，可能会影响性能。如果可能，可以将最后一层（输出层）的反向传播单独处理，以避免在循环中判断。
9. **可读性**：
   - 变量命名：有些变量名不够清晰（如`p1`, `p2`, `p3`）。在`backward`函数中，这些变量用于交换指针，但可以尝试用更清晰的名称或注释说明。
10. **初始化**：
    - 权重初始化：当前使用均匀分布初始化权重（0到1之间），然后乘以0.1。对于ReLU网络，通常使用He初始化（方差为2/n的均匀分布或正态分布）。偏置初始化为0或小常数。
11. **损失函数**：
    - 当前计算的是L1损失（绝对误差）。在分类任务中，通常使用交叉熵损失（Cross-Entropy Loss），它与Softmax结合更稳定且梯度更有利于学习。
12. **批归一化（Batch Normalization）**：
    - 可以考虑添加批归一化层，以加速训练并提高模型性能。
13. **代码复用**：
    - `forward`和`test_acc_loss`中的前向传播代码重复。可以提取一个公共函数。
14. **测试和验证**：
    - 在`test_acc_loss`函数中，测试时只使用了前`size`个样本（在main中调用时size=100）。通常，我们希望在整个测试集上评估。
15. **内存泄露**：
    - 程序在结束时没有释放所有分配的内存（例如在`model_initializer`中分配的内存）。虽然操作系统会回收，但良好的实践是释放。
16. **模块化**：
    - 将网络结构参数（如层大小、激活函数）通过配置文件或命令行参数传入，而不是硬编码在头文件中。
具体修改建议：
### 1. 改进softmax的数值稳定性（以forward.c为例）：
```c
// 原来的
double e_z[OUTPUT_LAYER_SIZE] = {0};
double sum = 0;
for (int j = 0; j < OUTPUT_LAYER_SIZE; j++) {
    e_z[j] = exp((z[HIDDEN_LAYER_COUNT][i][j]));
    sum += e_z[j];
}
// 改为：
double max_z = z[HIDDEN_LAYER_COUNT][i][0];
for (int j = 1; j < OUTPUT_LAYER_SIZE; j++) {
    if (z[HIDDEN_LAYER_COUNT][i][j] > max_z) 
        max_z = z[HIDDEN_LAYER_COUNT][i][j];
}
double sum = 0.0;
for (int j = 0; j < OUTPUT_LAYER_SIZE; j++) {
    e_z[j] = exp(z[HIDDEN_LAYER_COUNT][i][j] - max_z);
    sum += e_z[j];
}
```
### 2. 使用交叉熵损失：
当前在`train_acc_param_avg_abs.c`中计算训练损失是L1损失（绝对差），而在反向传播中（`backward.c`）输出层的梯度计算为：
```c
if (j == HIDDEN_LAYER_COUNT)
    dz[j][k] += (p1[k] = (p[i][k] - ((k == train_labels[(img_tag + i) % TRAIN_IMG_NUM]) ? 1 : 0)) / BATCH_SIZE);
```
这实际上是交叉熵损失结合Softmax的梯度（即预测值减去真实值）。因此，损失计算应该用交叉熵：
```c
// 在train_acc_param_avg_abs.c中，计算训练损失的部分：
train_loss = 0;
for (int i = 0; i < BATCH_SIZE; i++) {
    int label = train_labels[(img_tag + i) % TRAIN_IMG_NUM];
    // 交叉熵损失: -log(p[i][label])
    train_loss += -log(p[i][label]);
}
```
注意：由于p[i][label]可能为0，所以需要避免log(0)。可以加一个很小的数（如1e-10）：
```c
train_loss += -log(p[i][label] + 1e-10);
```
### 3. 权重初始化：
改为He初始化（适用于ReLU）：
```c
// 在model_initializer.c中
// 对于权重w，使用：
w[i][j][k] = (rand() / (double)RAND_MAX) * sqrt(2.0 / layer_array[i]); // 均匀分布
// 或者正态分布（但rand()产生均匀分布，可以用Box-Muller变换）
```
注意：He初始化是正态分布N(0, sqrt(2/n))，也可以用均匀分布：范围在[-sqrt(6/n), sqrt(6/n)]。
### 4. 避免临时内存重复分配：
在`backward`函数中，每次调用都分配两个`double`数组（大小为`LAYER_MAX`）。我们可以将其作为全局变量，或者在网络结构体中分配一次。
在`mlp.h`中声明：
```c
extern double *backward_temp1;
extern double *backward_temp2;
```
在`model_initializer.c`中分配：
```c
backward_temp1 = (double*)malloc(sizeof(double) * LAYER_MAX);
backward_temp2 = (double*)malloc(sizeof(double) * LAYER_MAX);
```
并在程序结束时释放。
### 5. 提取公共函数：
将前向传播过程提取为一个函数，供训练和测试使用。
### 6. 测试整个测试集：
在`test_acc_loss`函数中，将`size`改为`TEST_IMG_NUM`，并在main中调用时传入`TEST_IMG_NUM`。
### 7. 学习率衰减：
在`main.c`中，可以在每个epoch后调整学习率，例如：
```c
learning_rate[0][0] *= 0.99; // 隐藏层w
learning_rate[0][1] *= 0.99; // 输出层w
// 同理调整b的学习率
```
### 8. 释放内存：
在程序结束前（main函数返回前），添加一个函数释放所有动态分配的内存。

void parameter_update(void) {

    for (int i = 0; i < HIDDEN_LAYER_COUNT + 1; i++) {
        for (int j = 0; j < layer_array[i + 1]; j++) {

            }
        }
    }
}
